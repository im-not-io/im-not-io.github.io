User research

User interviews

User interviews are an important tool to help gauge features and improvements you want to add to your website. Unfortunately, the types of questions I asked were far too vague or off topic which resulted in responses that weren't as in-depth as I had hoped. Questions I asked queried about the desired features and concerns people had about the website, which helped a bit, but I should have created my own questions instead of using the stock questions provided. What I realized is that the quality of questions you ask will affect the quality of answers you will receieve. While my user interviews didn't persuade me to add any new features to my site, they helped convince me to leave certain features on my site. For example, a couple of the people I interviewed said I should include pictures so I ended up leaving in a feature I added on my site which swaps the page's background image every time the page is refreshed.

Competitor analysis

The competitor analysis, in the context of website design, is a method which examines various websites similar to the one you are building in hope of learning from those sites and using that new found knowledge to improve your own website. In my competitor analysis I tried to focus on other "personal websites"/portfolio websites. I reviewed the sites I reviewed because I wanted to get inspiration on how to design my website. For example, one website I examined was Ian Cinnamon's website who is also a programmer and shares some similarities with me. Ultimately, I ended up using a large background image like he did on his site. To help get a broad spectrum of inspiration I analysed a number of different sites including a software engineer/creative, an instructor, a digital designer, and a web designer/coder.

Feature value matrix

The feature value matrix is a tool which helps you determine which features are most important to your site and helps determine whether or not they should be included. I didn't really have a particular method for selecting the features I selected in my feature value matrix, I just kind of thought of things off the top of my head. What the feature value matrix is good for, however, is you can use it to get a sense of what really is feasible for the site. In the final version of my site I ended up only completing some of the things that had a better score on the feature value matrix. Some of the features I wanted, but scored poorly on the feature value matrix I was unable to implement. What makes the feature value matrix so useful and why I'd probably use it in the future is because it helps you focus on the most essential features your site really needs.

Feature prioritization

I prioritized features mainly on whether I thought the feature was important and whether or not I believed it was feasible within the time given. This feature prioritization is useful in determining which features are must-haves for the website. It also helps by visually showing the different features being considered for implementation and the order in which they should be implemented in. Sometimes even though a feature was a really cool idea, it ended up getting lower priority on the feature prioritization because it had a high technical effort level. For example, I wanted to do a video capture-and-upload application, but time didn’t permit that. Overall, the feature prioritization is a great tool for helping you manage the features you want to include on your website.

User stories

User stories are akin to a interaction design method in software engineering called personas, they help get a sense of the different types users and how they might use your software/website. The reason why user stories are important is because they allow us to identify our website's expected audience and how we can tailor our site to meet their needs. I tried to select a diverse set of people when choosing my stories including a teacher, a student, a hiring manager, a parent, a programmer, and just a general person browsing the web. By selecting all these different types of user stories I can get a feel for the audience of my website. For example, I realized a parent might not be as well-versed in using websites versus a college student and I hope to keep that in mind when designing my website.

Cognitive walkthrough

A cognitive walkthough is when you "walkthough" your website with someone and ask them various questions about the site and its functionality. A cognitive walkthrough is a great way of understand how real-life users will interact with your website. Out of all the activities I did the cognitive walkthrough was one of the most helpful since it gave me specific details that helped me improve my website. For example, the cognitive walkthoughs helped me realize some users had trouble realizing that my about me page had a drop down link that revealed content. The biggest challenge with cognitive walkthroughs in my opinion is actually meeting with people. I'm not very social so it's difficult me to communicate with people both in person or through video chat. Another issue I faced with the cognitive walkthroughs is time. I should have allotted for more time so I could get more detailed data on how to improve my site.

User testing

Summary about the types of tests conducted and why you chose them. Make sure to include any challenges faced. You can summarize the test results as well. Be short and to the point. I conducted two types of tests for user testing a clicktest and a question test. The clicktest was to get make sure the website was easy to navigate. The question test was just to help get data on how I can make the website better. One challenge I faced was people who didn't actually answer the questions. Another challenge was that I believe people took these tests very quickly and it's hard to say whether or not that may have created inaccurate results. Finally, I faced the challenge of divided opinions, where people's opinions on a particular feature were mixed which made it difficult to determine whether or not to keep that feature as is or modify/remove it. A high-level summary of my findings from user testing is as follows: overall people were able to navigate the website successfully, most people found the website visually appealing, most people didn’t think any links would open in a new window, some people didn’t realize there was hidden content on the about me page, people wanted a bigger font size and a brighter background, and there were mixed opinions on whether or not a changing background image was appealing or not.
